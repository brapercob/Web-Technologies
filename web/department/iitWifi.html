WiFi Service IITAP

WLAN / WiFi service at the Department of General Informatics (IITAP)

(More WiFi services at the University: MENET , eduroam )
 

Currently available access points in the Department of General Informatics:

 

    IITAP (1st floor, room 100 in the hallway)

    IITAP (at the door of laboratory 103 on the 1st floor in the corridor)

    IITAP (1st floor, room 108 in the hallway)

    IITAP (1st floor, room 107 / B)

    IITAP (2nd floor, laboratory 202)

Once connected, our device will automatically switch to the AP with the strongest signal ("wifi roaming").

Supported protocols: 802.11 / b / g / n

 

Access is done with the departmentâ€™s unique LDAP IDs , so there is no common shared key, everyone has unique access to the network.
Accesses are logged!
Outbound connections are logged, so in case of an incident, you can retrieve whose ID it happened!

 

The information required for setup is as follows:

Find the available Wifi points on your device called IITAP , and then in the connection settings you need to enter the following:

(On some operating systems, the order and appearance may differ. )

 

    Network authentication: WPA2-Enterprise

    Encryption: AES or TKIP

    EAP type: EAP-TTLS

    Password verification type: PAP

    Identifier name :  your LDAP account name

    Anonymous ID:  none, leave blank

    Password:  your LDAP account password

 

If possible, turn off certificate (CA) verification (or import this as a trusted CA).

 

EAP-TTLS is not supported by default on pre-MS Windows 8 operating systems . Fortunately, the WLAN interface driver replaces things in the WLAN stack so that EAP-TTLS becomes available e.g. In drivers for Intel and Broadcom interfaces. In other cases, SecureW2 must be installed. Help setting up Windows 10 .

 

IP assignment is done automatically (via DHCP), clients are given a private internal network address, which the server named hera sends from the NAT to the outside world.

Setup Help for Windows 10 (Windows 10 1703 or later)

 

0. If IITAP that is no longer working is added to the Wifi list, it will be deleted:
Engine -> Network and Internet -> Wi-Fi -> Manage known networks -> IITAP -> Forget



1. Add the IITAP network to the system:
1.a Control Panel -> Network and Internet -> View network status and network tasks ->
1.b Set up a new connection or network -> Manually connect to a wireless network ->


Network name : IITAP
Security type : WPA2-Enterprise

"Connect even if the network does not publish its name (SSID)" : YES, be checked Do not close the window

after pressing the [Next] button !



2.a Change connection settings -> "Security" tab -> "Choose your network authentication method" ->


Microsoft: EAP-TTLS

Below it is
"Remember credentials for connection every time I log in" : YES, check

2.b and next to " Settings " button ->
"Enable identity protection" : NO, ie not checked
"Don't ask the user if you can't authenticate server " : NO, ie unchecked

Click [OK] to save the changes!



3. Select "Advanced Settings" ->


"Specify authentication mode" -> "User authentication"

then select  "Save credentials" :

Enter your iit LDAP account name and password here and save.



4. Click [OK] to close all windows, then press [Close] to add the network.

 

5. You can then connect to the network as usual (by clicking the wifi signal next to the clock from the taskbar and then selecting the IITAP connection).

During the first connection, you will be asked for a confirmation to approve!
